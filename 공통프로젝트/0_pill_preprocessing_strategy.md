# 알약 이미지 전처리 전략

## 프로젝트 목표

식품의약품안전처 알약 이미지 약 27,000장을 활용하여 다음 3개의 모델을 구축한다.

1. 모양 분류 모델
2. 색상 분류 모델
3. 문자/마크 인식 모델

사용 모델 예시

- EfficientNet-B0 (모양 분류)
- EfficientNet-B0 (색상 분류)
- CNN/OCR 기반 모델 (문자 및 마크 인식)

---

# 원본 데이터 특징

## 이미지 구성

원본 이미지에는 다음 정보가 포함된다.

- 알약 앞면
- 알약 뒷면
- 배경색
- 규격선
- 치수 정보
- 식약처 로고
- 텍스트 정보

예시

```
-------------------------
| 알약 앞면   알약 뒷면 |
-------------------------
```

또는

```
-------------------------
|      알약 앞면        |
|            알약 뒷면  |
-------------------------
```

---

# 전처리 목표

원본 이미지에서

- 규격선 제거
- 치수 제거
- 식약처 로고 제거
- 텍스트 제거

후

각 알약을 개별 이미지로 저장한다.

예시

```
원본
 ├── 앞면
 └── 뒷면

↓

결과

pill_001_front.png
pill_001_back.png
```

---

# 크롭 방식 결정

## 채택 방법

OpenCV 기반 Contour 검출

절차

```
원본 이미지
    ↓
배경 분리
    ↓
Contour 검출
    ↓
면적 기준 객체 추출
    ↓
알약 2개 선택
    ↓
개별 Crop
```

---

## 선택 이유

### 장점

- 별도 학습 필요 없음
- 27,000장 일괄 처리 가능
- 매우 빠름
- 구현 난이도 낮음

### 단점

- 일부 특이 케이스 존재 가능
- 실패 이미지 검수 필요

---

# 채택하지 않은 방법

## 단순 가로/세로 절단

문제

- 사선 배치 존재
- 알약 위치가 일정하지 않음

따라서 사용하지 않음

---

## YOLO

장점

- 높은 정확도

단점

- 라벨링 필요
- 현재 데이터에서는 과도한 방법

필요 시 실패 이미지에만 적용

---

## SAM (Segment Anything)

장점

- 정확도 높음

단점

- 처리 속도 느림
- 27,000장 일괄 처리 부담

필요 시 예외 처리용으로 사용

---

# 가장 중요한 원칙

## 알약 세그멘테이션 금지

다음 방식은 사용하지 않는다.

```
배경 제거
↓
알약만 추출
↓
PNG 저장
```

---

이유

다음 정보가 손실될 수 있음

- 각인 문자
- 마크
- 분할선
- 그림자
- 반사광
- 입체감

특히 연질캡슐에서 손실이 큼

---

# 최종 저장 방식

## 권장 방식

Bounding Box Crop

```
배경 일부 포함
    ↓
알약 Crop
    ↓
저장
```

---

## 여백 유지

알약 경계와 이미지 경계가 붙지 않도록 한다.

권장

```
알약 크기 기준

상하좌우
10~20%
Padding
```

예시

```
원본 객체

100 x 50

↓

Padding 추가

120 x 70
```

---

# 해상도 정책

## 원본 보존

크롭 직후

Resize 하지 않는다.

예시

```
Crop
↓
PNG 저장
```

---

## 이유

문자/마크 모델에서

- 각인
- 로고
- 분할선

정보가 중요하기 때문이다.

# 최종 결론

현재 식약처 알약 이미지 데이터셋은

- 배경과 알약의 구분이 명확하고
- 대부분 알약 2개가 분리되어 있으며
- OpenCV Contour 기반 검출이 충분히 가능하다.

따라서

**"OpenCV Contour 기반 자동 크롭 + 여백 유지 + 원본 품질 보존"**

전략을 채택한다.

세그멘테이션은 수행하지 않으며,

YOLO 및 SAM은 예외 이미지 처리용 보조 수단으로만 사용한다.
